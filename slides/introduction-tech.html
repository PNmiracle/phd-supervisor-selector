为什么需要
这个Skill？

博士选导的三大痛点

🏰

信息太分散

100 所大学 = 100 种网站系统。导师信息散落在各个院系页面，没有一个统一入口。

⏰

人工太耗时

手动搜一个学校就要 1-2 小时——点开每个个人页、读研究方向、判断是否带博、记录链接。

🤷

判断太难

教学岗还是科研岗？退休了吗？研究方向真的匹配吗？——没有领域知识很难判断。

💡
场景：一个博士生要申请，需要从全球上百所大学里找到研究方向匹配、有招生名额、能带博士的导师——这个工作量，自己干要好几周。
03 / 28

它不是搜索引擎
是研究助理

关键词搜索只能"返回可能有关系的页面"，这个 Skill 会"判断、验证、整理"

🔍

搜索引擎

phd-supervisor-selector

理解研究方向 → 多策略搜索 → 逐页验证 → 判断匹配度 → 填入结构化表格。给你的是一个可以直接用的导师名单。

🕸️

能批量抓取，但遇到 Cloudflare 就死，不理解研究方向含义，不会判断匹配度。

06 / 28

搜索策略：五招
从温柔到暴力

根据每个网站的性格，自动选择最合适的策略

🚪 第一招

直接访问 · curl 请求

伪装浏览器身份直接访问 staff 页面，解析 HTML 提取人名链接。适合老老实实服务器渲染的站。

🗝️ 第二招

JS 探针 · API 挖掘

下载 JS 文件，搜索 api、endpoint、baseUrl——绕过网页壳子直接拿 JSON 数据。

🔎 第三招

搜索引擎按名索骥

拿到导师名+学校 → 搜索引擎搜 "[名字] [学校] professor" → 获取真实存活 URL。

🖥️ 第四招

浏览器模拟人验证

curl 拿不到的 JS 渲染页面 → 启动浏览器模拟人点开页面、点击隐藏 tab、截图确认内容。Cloudflare 盾的克星。

🤖 第五招

智能分配·多个子智能体并行处理

同时分配多个 AI 子智能体分头搜索不同学校，最后汇总——
速度翻 5-10 倍。

07 / 28

不是搜到就停
而是越挖越深

Deep Discovery Pipeline —— 每个学校挖到底

🔁

按需回查

不在每轮搜索中自动重试 404——只在用户要求或有信号（新学年、网站改版）时才回查，避免浪费 token。

🌿

跨院系联想

不只搜"服装系"，还搜人类学、文化研究、设计史——导师可能藏在隔壁院系。

🏷️

隐藏标签点击

自动点击页面隐藏 tab（Research Interest、Publications）——关键信息常藏在折叠面板后。

🧪

备用域名探测

主站打不开？试试研究生院、研究门户子域名——同一大学不同系统防护等级不同。

📊

API 批量拉取

找到 API 后一次拉全部数据到本地筛——不翻页不逐页爬，效率高几十倍。

遇到盾牌怎么办？
不放弃，也不死磕

Cloudflare 反爬、JS 渲染壳、403 拦截……每种都有对应策略

🛡️ Cloudflare

换路径

主站被拦 → 试试 Pure 研究门户、研究生院、ePrints。同校不同系统防护不同。

💻 JS 渲染壳

找 API

下载 JS 搜 api、endpoint、baseUrl——找到真正的数据接口，绕过壳子拿 JSON。

🚫 全部失败

标记⚠️不删

彻底打不开的保留在表中标注 "⚠️需手动验证"——不让任何候选人遗漏。

🏋️
核心韧性：搜索引擎优先找 URL → curl 验证 → 浏览器模拟人确认内容 → JS 渲染站挖 API → 全失败标记 ⚠️。不删任何候选人，学生知道哪些需要手动确认。

四层竞争壁垒

从技术到数据，每一层都让抄袭更难

🔧

第 1 层：技术壁垒

多策略搜索 + 反爬对抗 + 浏览器验证——不是简单的"写个爬虫"，而是针对每家大学定制策略的智能系统。

🗄️

第 2 层：数据壁垒

搜得越多，积累的验证过的导师画像越多。后来者即使有同样的技术，没有历史数据也推荐不准。

🧠

第 3 层：领域知识壁垒

知道什么研究方向匹配、什么导师有风险、什么标题意味着教学岗——这需要真实的学术背景和大量实践积累。

🔄

第 4 层：网络效应壁垒

学生用 → 反馈数据 → 推荐更准 → 更多学生用 → 中介接入 → 数据更多 → 飞轮自转。

今天就可以开始的事

不等完美产品——现在就能验证需求、积累资产

📝 第 1 周

建内容账号

小红书/知乎注册"PhD选导助手"。发 3-5 篇选导案例帖——把 Skill 搜出来的好结果截屏打码发上去。

🤝 第 2 周

找 10 个内测用户

从评论/私信里挑 10 个真实申请者，免费帮搜。收集反馈：哪里不够好？还想要什么功能？

🛠️ 第 3-4 周

搭 MVP 骨架

买个域名 · 搭个 Landing Page（介绍+邮箱收集）· 接 Stripe 支付链接。不写代码，用 no-code 也能跑。

📊 第 2 个月

付费验证

10 个付费用户（哪怕 $9 尝鲜价）→ 证明有人愿意付钱。$29 搜一次太贵？试试 $9 限时价测转化。

🔄 第 3 个月

迭代 + 招聘

根据反馈迭代。攒够 50 个付费用户 → 找技术合伙人/外包搭正式产品。同时保持内容账号持续输出。

💡
核心理念：先验证再建。Skill 就是你的 MVP——先用它服务真实用户，确认有人付费，再投入精力做产品。
