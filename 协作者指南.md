# 协作者快速指南

> 本文档给**协作者（小白）**看，管理员不用看。
> 目标：让你能安全地和仓库管理员一起维护这个 Skill，不出错。

---

## 一句话记住流程

```
改文件 → 对 AI 说"同步技能" → 等 AI 跑完 → 把 PR 链接发给管理员
```

**就这三步，不需要懂 git。**

---

## 第一次使用（只需做一次）

### 第1步：Clone 仓库

**对 AI 说：**
> "帮我把 `PNmiracle/phd-supervisor-selector` 克隆到 `~/.workbuddy/skills/` 目录下"

AI 会自动执行：
```bash
cd ~/.workbuddy/skills/
git clone https://github.com/PNmiracle/phd-supervisor-selector.git
```

**验证成功：** 你的电脑上会出现 `~/.workbuddy/skills/phd-supervisor-selector/` 这个文件夹。

---

### 第2步：配置 git 身份

**对 AI 说：**
> "帮我配置 git，用户名填 `你的GitHub用户名`，邮箱填 `你的GitHub注册邮箱`"

---

## 日常使用（每次都这样）

### 场景A：我想改一个文件（比如加学校策略）

**第1步：改文件**
- 用 VS Code 或任何编辑器打开 `~/.workbuddy/skills/phd-supervisor-selector/references/school-strategies.md`
- 改完，**保存**

**第2步：对 AI 说"同步技能"**

> "同步技能"

AI 会自动执行：
1. 展示你改了哪些文件（让你确认）
2. 拉取最新代码（保证你本地是最新的）
3. 开一个新分支
4. 提交你的改动
5. 推送到 GitHub
6. 自动创建 PR（Pull Request）
7. 输出 PR 链接

**第3步：把 PR 链接发给管理员**

AI 会输出类似这样的链接：
```
✅ PR 已创建：https://github.com/PNmiracle/phd-supervisor-selector/pull/12
等待仓库管理员合并～
```

**复制这个链接，微信发给管理员就行。**

---

### 场景B：我想改多个文件

**和场景A一样**，改完所有文件后，对 AI 说"同步技能"就行。AI 会自动把所有改动都提交。

---

### 场景C：我只想提交部分文件（其他文件先不改）

**对 AI 说：**
> "同步技能，但只提交 `school-strategies.md`，其他文件先不提交"

AI 会只提交你指定的文件。

---

## 管理员合并后，我怎么同步最新代码？

**下一次你要改文件前，对 AI 说：**
> "同步技能"

AI 会自动先拉取最新代码（包括管理员合并的内容），保证你本地是最新的。

**不需要额外操作。**

---

## 如果 AI 说"有冲突"（不用慌）

偶尔会出现这种情况：
```
⚠️ 冲突检测：以下文件有冲突：
  - references/school-strategies.md
请决定保留哪个版本，或两边都在。
```

**怎么办：**

1. AI 会展示冲突内容（带 `<<<<<<<` 标记）
2. 你告诉 AI："保留我的版本" 或 "保留远程的版本" 或 "两边都在"
3. AI 会自动解决冲突，继续提交

**冲突不可怕，AI 会帮你处理。**

## 常见问题

### Q1：我改完文件，忘了说"同步技能"就关了 AI，怎么办？

**没关系**，下次打开 AI，直接说"同步技能"就行。你的改动还保存在本地文件里。

---

### Q2：我能直接在网上改文件吗？

**可以**，但推荐用本地编辑器（VS Code 等），因为：
- 本地编辑器有语法高亮，不容易改错
- 可以一次改多个文件，再一起提交

如果实在想在网页上改，参考[网页版操作流程](#网页版操作流程可选)。

---

### Q3：PR 创建后，管理员一直没合并，我能再改别的文件吗？

**可以。** 说"同步技能"会开一个新的 PR，不影响之前的 PR。

管理员合并时，会按 PR 顺序一个个合并。

---

### Q4：我想看我之前提交过哪些 PR？

**对 AI 说：**
> "帮我看看我开过哪些 PR"

AI 会列出所有 PR 的状态（开放中 / 已合并 / 已关闭）。

## 网页版操作流程（可选）

如果你不想用本地编辑器，也可以直接在 GitHub 网页上改：

1. 打开 `https://github.com/PNmiracle/phd-supervisor-selector`
2. 找到要改的文件（比如 `references/school-strategies.md`）
3. 点文件右上角的 **✏️ 铅笔图标**（Edit this file）
4. GitHub 会提示 **"Create a new branch for this commit"**（勾上它）
5. 改完，页面底部填 **Commit description**，点 **"Propose changes"**
6. 自动跳到开 PR 页面，点 **"Create pull request"**
7. 把 PR 链接发给管理员

## 紧急情况处理

### 情况1：我不小心把文件改乱了，想恢复

**对 AI 说：**
> "帮我把 `school-strategies.md` 恢复到最新版本"

AI 会执行 `git restore`，把文件恢复到最近一次提交的状态。

---

### 情况2：我说"同步技能"时报错了

**直接把错误信息截图发给管理员**，或者把错误信息复制给 AI，AI 会根据 `SKILL.md` 里的指引处理。

---

### 情况3：我想删掉某个文件

**对 AI 说：**
> "帮我把 `old-file.md` 删掉，然后同步技能"

---

## 总结：你需要记住的命令只有一句

| 场景 | 对 AI 说的话 |
|------|----------------|
| 改完文件，想提交 | **"同步技能"** |
| 想看 PR 状态 | **"帮我看看我开过哪些 PR"** |
| 文件改乱了，想恢复 | **"帮我把 XX 文件恢复到最新版本"** |

**就这三句，够了。**
