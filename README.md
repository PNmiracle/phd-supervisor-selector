# PhD Supervisor Selector · 博士选导助手

面向留学机构，为不同方向的学生快速建立可核验的博士导师列表。搜索各大学官方页面，判断导师是否具备 PhD 指导资格，直接写入 Vika 在线表格。不限学科方向。

---

## 核心能力

| 能力 | 说明 |
|------|------|
| **L0-L4 搜索框架** | 5 层访问策略，覆盖静态站、SPA 壳、WAF 封锁、API 发现、搜索引擎兜底——学科无关 |
| **学校策略库** | 60 所学校（UK 15 / AU 9 / HK 8 / SG 3 / Macau 2 / 中外合办 7 / NZ 5 / EU 4 / US-CA 6），每所记录已知访问层、URL 模式、失败方法 |
| **分地区筛选规则** | UK Lecturer 可带博、美国 Lecturer 不行——按地区和职称体系判定指导资格 |
| **Vika 零依赖 CRUD** | Python 标准库直调 REST API，无需安装 SDK；选导意向保护、批量去重、审计脚本 |
| **备注风格规范** | 像面对面和学生说话的语气；匹配度分级标识（🔥/👍/👀），禁止机器打分式评价 |
| **学习闭环** | "结束学生"自动提取经验 → 更新策略库 → 提交 GitHub，用着用着就变强 |

## 快速开始

### Vika 模式（推荐）

> 你给 Vika 链接 + API Token，AI 直接在维格表里增删改查。

| 步骤 | 你说的 |
|------|--------|
| ① 安装 Skill | `帮我安装这个skill，https://github.com/PNmiracle/phd-supervisor-selector` |
| ② 给 Vika 链接 | `打开这个表 https://vika.cn/share/shrXXX/dstXXX/viwXXX` |
| ③ 给 API Token | `我的 token 是 uskXXX` |
| ④ 直接操作 | `列出所有导师` / `新增一条XXX` / `把Department翻译成中文` / `检查重复` |

**Token 安全**：不要在聊天框贴 token，跑一行命令一劳永逸：

```bash
echo 'export VIKA_TOKEN=你的token' >> ~/.zshrc && source ~/.zshrc
```

### Excel 模式（可选回退）

没有 Vika 表时，拖入 Excel，AI 搜索学校官网生成导师表。需要 `openpyxl`。

## 使用阶段

```
Phase 1 — 新建 / 导入
  │   Vika: 给链接 + Token，AI 列出表结构，自然语言 CRUD
  │   Excel: 说方向+学校，AI 建表并搜索填充
  │
Phase 2 — 深度搜索
  │   按学校搜索官网师资页、博士项目页
  │   自动并行子 Agent 加速（多学校同时搜）
  │   L0-SPA → L1 curl → L2 API → L3 搜索引擎 → L4 Tavily 逐层升级
  │
Phase 3 — 清洗 & 补全
  │   Department 翻译成中文
  │   补全缺失的 博士申请信息 / 其他导师信息 链接
  │   去重、标记空行、运行审计脚本
  │
Phase 4 — 持续维护
  │   自然语言操作："把XXX的状态改成已套磁"
  │   新增导师、删除放弃的
  │   "结束学生" → 自动更新策略库 + 提交 GitHub
```

## 备注风格

备注是给学生看的，语气要自然、有"人味"，像坐在旁边帮 ta 梳理信息。

**语气要求：**
- 不要写"符合XX方向""匹配XX方向"——太机械
- 要写"可以往XX方向贴合""和XX方向能搭上"——有商量感，留余量
- 禁止使用 `高度匹配`、`很匹配`、`完美匹配` 等机器打分式用语

**匹配度分级（备注结尾标识）：**

| 级别 | 标识语 | 触发条件 |
|------|--------|----------|
| 强匹配 | `特别推荐看一下～` | 方向多重重叠 + 顶刊发表 + 正在招博士 |
| 一般匹配 | `比较相关～` | 研究方向与学生 1-2 个目标方向直接相关 |
| 弱相关 | `可以备选一下哈～` | 方向有交叉但非核心对应 |

**示例：**
- `副教授；消费者判断与决策、跨期选择，和品牌传播能搭上。`
- `助理教授；消费者决策、道德决策、自我概念清晰度；MIT PhD，可以关注一下。`
- `教授；做数字消费者行为和 AI 在营销里的应用，品牌传播和媒介技术都能搭上；Nature和JMR都有发，正在招博士生。特别推荐看一下～`

## 多人协作

> 📖 协作者请读 [`协作者指南.md`](协作者指南.md) · 管理员请读 [`管理员指南.md`](管理员指南.md)

两句话记住流程：

| 场景 | 对 AI 说 |
|------|-----------|
| 每次开始搜学生前 | **"同步技能"** — 拉最新版，保证用的是最新策略 |
| 搜完一个学生后 | **"结束学生"** — 提取经验存回去，让下次更准 |

协作者不需要懂代码、不需要懂 git，全程用自然语言跟 AI 聊。首次设置只需配 git 身份 + 克隆仓库，详见协作者指南。
